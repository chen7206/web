## Markdown개념:   
```
마크다운(markdown)은 일반 텍스트 문서의 양식을 편집하는 문법이다. README 파일이나 온라인 문서
혹은 일반 텍스트 편집기로 문서 양식을 편집할 때 쓰인다. 마크다운을 이용해 작성된 문서는 쉽게
HTML 등 다른 문서형태로 변환이 가능하다.
```
       
             
## Markdown역사:
```
존 그루버는 2004년에 문법 면에서 에런 스워츠와 중대한 협업을 통해 마크다운 언어를 만들었으며
사람들이 읽기 쉽고 쓰기 쉬운 플레인 텍스트 포맷을 사용하여 쓸 수 있으면서 구조적으로 유효한
XHTML(또는 HTML)로 선택적 변환이 가능하게 하는 것이 목표이다.
```
      
            
## Markdown문법:
    
### (1)#문법
       
- 제목 수준을 지정할 때 사용한다. 
- # 한개를 쓰면 <h1> 태그 수준을, 두개를 쓰면 <h2> 태그 수준을 의미하고
  갯수가 늘어날 수록 제목 수준이 내려간다. (글씨체, 글씨크기의 변화)
- 총 6개까지 붙여서 사용할 수 있다. 

     
*예시 코드*
```
###### 6번 중첩
##### 5번 중첩
#### 4번 중첩
### 3번 중첩	
## 2번 중첩
# 1번 중첩
```
*결과*
      
###### 6번 중첩
##### 5번 중첩
#### 4번 중첩
### 3번 중첩	
## 2번 중첩
# 1번 중첩
     
***     
       
***          
### (2)리스트
   
- "-", "+", "*" 이후 텍스트를 입력시 자동으로 리스트로 변환된다. 
 리스트 문자 이전에 공백이나 탭스페이스 여부에 따라 자동으로 리스트 수준이 결정 된다. 
       
***       
### (3) 번호 있는 리스트
       
- 일반 리스트와 같이 텍스트 앞에 "1." 형식으로 적어주면 된다. 
- 번호 수준 역시 공백에 따라 자동 변환된다.
      
***           
### (4) 텍스트 
       
- 문서의 본문에 사용 
- 사용법 : 그냥 적으면 됨
      
*예시 코드*
```
이것은 텍스트 입니다.
```
   
*결과*
     
이것은 텍스트 입니다.
      
***            
### (5) 강조
      
- 텍스트를 강조할 때 사용
- 내용을 감싸주면 됨
- 사용법 : * or _ or ** or __ or ~~

      
*예시 코드*
     
```
*Italic*
_Italic_
 **Bold**
 __Bold__
 ++Under Line++
 ~~Strike Out~~
```
     
*결과*
  
 *Italic*
_Italic_
 **Bold**
 __Bold__
 ++Under Line++
 ~~Strike Out~~   
      
***           
 ### (6) 수평선 
        
 - 문장을 구분할 때 사용.
 - 2가지 방법 모두 같은 결과이다.
 - 사용법 : --- or *** or ___

       
*예시 코드*    
```
  ***
  ---
   
```
*결과*
     
***
---
    
***
### (7) 인용구
- 이메일에서 사용하는 인용구
- 인용구안에 다른 요소들을 포함할 수 있음
- 사용법 : >
     
*예시 코드*  
```
> 인용구의 내용
> ###### 헤더를 포함해 봤습니다.
```
     
     
*결과*
      
> 인용구의 내용
> ###### 헤더를 포함해 봤습니다.  
     
          
***
### (8) 코드
     
- 특정 부분만 코드 블록으로 사용 가능
- 혹은 강조용으로 사용하기도 함
- 사용법 : ' (!에서 Shift땐 키)


*예시 코드*
  
```
`# 이렇게도 사용하고`
`강조`도 가능합니다`
```
     
     
*결과*
     
`# 이렇게도 사용하고`
`강조`도 가능합니다.
    
         
***

### (9) 목록
      
- 특정 줄 강조 혹은 목차등을 표시할 때 사용
- 오름차순으로 요소들을 표현할 때도 가능
- 순서있는 목록 사용법 : 1 or 2 or 3 - 잘못 입력해도 알아서 수정됩니다.
- 순서없는 목록 사용법 : * or + or - 셋중 아무거나 사용하면 됩니다
     
         
         
  
*예시 코드*
    
```
1. 1번 요소
3. 3번 요소
2. 2번 요소

  * 제목
    * 부제목
      * 내용
```
   
   
*결과*
    
 1. 1번 요소
3. 3번 요소
2. 2번 요소

  * 제목
    * 부제목
      * 내용   
          
                
***
### (10) 표 [Table]
     
- 표를 나타냄
- 안에 다른 요소를 포함 할 수 있음
- 사용법 : [:-] 오른쪽 정렬 | [:-:] 중앙 정렬 | [-:] 왼쪽 정렬
    
    
    
    
*예시 코드*
     
```
| 이름 | 생년월일           |성별| 
| :-   | :-:               | -: | 
| chenhui | 1993-03-17 | 남 | 
```
   
*결과*  
   
| 이름 | 생년월일           |성별| 
| :-   | :-:               | -: | 
| chenhui | 1993-03-17 | 남 | 
     
          
***
